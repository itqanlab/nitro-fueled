## Project Conventions

From CLAUDE.md:
- Git: conventional commits with scopes (e.g., `refactor(TASK_2026_058):`)
- Task states: `CREATED | IN_PROGRESS | IMPLEMENTED | IN_REVIEW | COMPLETE | FAILED | BLOCKED | CANCELLED`
- Agent files are markdown with YAML frontmatter
- Skill files are markdown with structured sections
- Command files are markdown

From review-general.md relevant rules:
- **TypeScript**: No `any` type, no `as` assertions, explicit access modifiers, string literal unions for status fields
- **Naming**: kebab-case files, camelCase functions, PascalCase interfaces
- **Error handling**: Never swallow errors; `writeFileSync`/`readFileSync` without try/catch propagates raw OS error
- **File generation**: guard all I/O with try/catch; validate body non-empty before writing
- **Two maintenance paths conflict**: when both CLI and human agent can modify same file, need coordination contract
- **Enum values must match canonical source**: status values must match exactly — `COMPLETE` vs `COMPLETED`, `|` vs `/`
- **Partial delegation defeats SOT pattern**: if removing inline rules in favor of external file, remove ALL inline copies
- **Step numbering must be flat and sequential**: no mixed schemes like "Step 5, Step 5b"
- **Cross-file references must use names not numbers**: use descriptive section names that survive renumbering
- **New status/enum values must be added to canonical reference first**: `FIXING` added to `VALID_STATUSES` — verify it's in task-template.md/registry Status table
